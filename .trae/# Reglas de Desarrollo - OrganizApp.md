# Reglas de Desarrollo - OrganizApp

Este archivo define las normas de trabajo para el asistente (Trae) y los desarrolladores.

## 1. Documentación Viva
*   **Actualización Obligatoria de Módulos**: Cada vez que se implemente un cambio, se refactorice código o se modifique la lógica de negocio de un módulo a través de un prompt, es **obligatorio** actualizar el archivo de descripción correspondiente en `.trae/modules/`.
*   La documentación debe reflejar siempre el estado actual del código y no quedarse obsoleta.

## 2. Estructura del Proyecto
*   Mantener la separación de responsabilidades.
*   Seguir las descripciones de arquitectura definidas en los archivos de módulos.

---
*Nota: Este archivo debe ser consultado por Trae antes de realizar cambios estructurales importantes.*
